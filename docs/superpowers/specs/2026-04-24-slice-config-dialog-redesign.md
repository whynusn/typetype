# 载文设置 Dialog 重构设计

## 背景

当前 `SliceConfigDialog.qml` 的重打条件仅支持单一指标+比较符+阈值（如"速度 < 60"）。用户需求将其扩展为四指标联合判定，并增加乱序、开始片段等能力。

## 目标

1. 重构合格指标为固定四指标（击键、速度、键准、达标次数），仅设下界，逻辑与连接
2. 增加"未达标或有错字时"事件选择（乱序(重打) / 重打 / 无）
3. 增加"开始片段"输入（默认1）
4. 增加全文乱序按钮
5. 整体优化 Dialog 布局与交互

## 方案：紧凑列表式（方案A）

### 合格指标区域

四个条件固定展示，每行格式：`[指标名]  ≥  [输入框]  [单位]`

| 指标 | 默认下界 | 单位 | 校验范围 |
|------|---------|------|---------|
| 击键 | 200 | 次/秒 | 1~999 |
| 速度 | 100 | CPM | 1~999 |
| 键准 | 95 | % | 0~100 |
| 达标次数 | 1 | 片 | 1~9999 |

- `≥` 为固定文本，不可修改
- 四个条件需**同时满足**才算"达标"
- 任一不满足（或达标次数累计不够）即触发下方设定的事件

**"达标次数"语义**：历史累计中，其余三项指标均达标的片数（含当前片）。例如设置达标次数=3，则连续3片都满足击键/速度/键准下界后，第3片才算真正达标并进入下一片；否则触发事件。

### 事件设置区域

紧跟合格指标下方：

```
未达标或有错字时:  [乱序(重打) ▼]
```

选项：
- `乱序(重打)` — 未达标时乱序当前片段并重新加载
- `重打` — 未达标时原样重新加载当前片段
- `无` — 不触发任何动作，直接进入下一片（相当于忽略所有指标）

### 分片设置区域

```
每片字数: [30]    开始片段: [1]    ☑ 全文载入（不分片）
```

- 新增"开始片段"输入框，默认1，范围 1 ~ 总片数
- 开启"全文载入"时，两个输入框都禁用

### 乱序按钮位置

放在**文本内容 Frame 的标题行右侧**。用户看到文本内容后，可直接点击乱序当前文本。

按钮样式：小型文字按钮（"乱序"），与标题同行右对齐。

### Footer

保持两个按钮：取消 | 开始载文

### Dialog 尺寸调整

当前高度 620，增加合格指标行数后预计需要 **680~720**。

## 数据流变更

### QML → Bridge

`setupSliceMode` 签名从：
```
setupSliceMode(text, sliceSize, retypeEnabled, metric, operator, threshold, shuffle)
```

变更为：
```
setupSliceMode(text, sliceSize, startSlice,
               keyStrokeMin, speedMin, accuracyMin, passCountMin,
               onFailAction, shuffle)
```

其中 `onFailAction` 枚举：`"shuffle_retype" | "retype" | "none"`

### Bridge → SessionContext

`setup_slice_mode` 参数同步变更，移除单一 `metric/operator/threshold`，替换为四个下界 + 事件动作。

`should_retype()` 逻辑从单一条件判断改为：
1. 检查当前片 stats 是否满足 击键≥、速度≥、键准≥
2. 若满足，累计达标次数+1
3. 若累计达标次数 < passCountMin，返回 True（需要重打/事件）
4. 若任一项不满足，返回 True
5. 全部满足且达标次数够，返回 False（不重打，进入下一片）

### 错字触发事件

当前已有错字统计。用户在打字过程中产生错字时，也视为"未达标"的一种，在打完当前片后同样触发 `onFailAction`。

是否需要**立即**触发（打字中途发现错字就中断）还是**打完该片后**触发？

**决定**：打完当前片后统一判定。这样与达标次数的累计逻辑一致，且不打断用户当前打字流。

## 校验规则

所有输入框在失去焦点或点击"开始载文"时进行校验：

- 击键/速度/达标次数：整数，下界1，上界999
- 键准：整数，0~100
- 每片字数：整数，1~文章字数
- 开始片段：整数，1~总片数（若已知文本长度）

## 测试要点

1. 四个指标单独达标/不达标组合是否正确触发事件
2. 达标次数累计逻辑（跨片累计）
3. 开始片段>1时，第一片直接跳到对应片段
4. 乱序按钮影响文本内容，且不影响其他设置
5. "无"事件模式下，即使不达标也正常推进
6. 所有输入框边界值校验
